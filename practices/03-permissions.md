# Разрешения и безопасность

## Суть
Файл .claude/settings.local.json — настройка доступа AI к инструментам для каждого проекта. Этот файл находится в папке .claude/ внутри вашего проекта. Его можно открыть любым текстовым редактором или попросить Claude Code показать содержимое.

## Уровни доступа

### Максимальный (активная разработка)
Bash, WebSearch, WebFetch, Edit, навыки — широко. Для проектов, где AI активно пишет и запускает код.

### Ограниченный (инфраструктура)
Только SSH, специфичные домены для WebFetch. Для работы с сетевым оборудованием, серверами.

### Минимальный (аналитика)
Без Bash, без записи файлов. Только чтение и конкретные домены. Деструктивные команды исключены на уровне настроек.

## Известная ловушка: массивы permissions не объединяются

Проектный файл `.claude/settings.local.json` с массивом `permissions.allow` **полностью перекрывает** глобальный `~/.claude/settings.json`. Массивы НЕ объединяются — проектный заменяет глобальный целиком.

**Что это значит:** если глобально разрешены `Bash(git *)`, `Bash(npm *)`, а в проекте вы добавили только `Bash(docker *)` — git и npm станут запрещены в этом проекте.

**Решение:** в проектном `settings.local.json` дублируйте все нужные глобальные разрешения плюс проектные.

**Диагностика:** если Claude Code начал запрашивать разрешения на команды, которые раньше были разрешены — проверьте проектный settings.local.json.

## Самооптимизация разрешений
AI может проанализировать историю команд и предложить оптимизацию. Например, десятки отдельных правил для однотипных команд можно обобщить в один паттерн — AI сам предложит консолидацию.

## Подготовка к Silent Mode
Перед автономной работой:
1. Попросить AI отрефакторить текущий список разрешений
2. Проанализировать, какие команды реально используются
3. Спрогнозировать, какие команды понадобятся для задачи
4. Одобрить изменения → AI готов работать автономно без запросов

## Принцип
Минимум доступа по умолчанию, расширение по мере необходимости.
