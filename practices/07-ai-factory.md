# AI Factory — навыки и конвейер

## Суть

AI Factory — это подход к работе с AI, при котором весь жизненный цикл задачи покрыт навыками (skills). Вместо хаотичных промптов — структурированный конвейер: исследование → планирование → реализация → проверка.

Существует готовый open-source инструмент **[AI Factory](https://github.com/lee-to/ai-factory)**, который реализует этот подход «из коробки». Он устанавливается одной командой и сразу даёт полный набор навыков для осознанной разработки.

## Инструмент AI Factory

**Что это:** CLI-инструмент, который автоматически настраивает навыки, патчи и workflow для вашего проекта. Поддерживает Claude Code, Cursor, Windsurf, Roo Code и другие AI-агенты.

**Установка:**
```bash
npm install -g ai-factory
ai-factory init
```

При инициализации AI Factory:
- Определяет ваш AI-агент
- Устанавливает подходящие навыки
- Настраивает MCP-серверы (если агент поддерживает)
- Создаёт структуру `.ai-factory/` в проекте

**Ключевые команды:**

| Команда | Что делает |
|-|-|
| `/aif-explore` | Исследование: изучить варианты перед планированием |
| `/aif-plan` | Создать план задач с зависимостями и чекпоинтами |
| `/aif-improve` | Улучшить существующий план |
| `/aif-implement` | Реализация по плану с коммитами на чекпоинтах |
| `/aif-fix` | Исправление ошибки + автоматический патч для обучения |
| `/aif-evolve` | Улучшение навыков на основе накопленных патчей |
| `/aif-docs` | Автогенерация документации |
| `/aif-ci` | Генерация CI/CD пайплайнов |

## Самообучающаяся система патчей

Главная ценность AI Factory — система патчей. После каждого `/aif-fix` создаётся файл в `.ai-factory/patches/`:

```markdown
# Null reference в компоненте без данных

Problem: TypeError при рендере
Root Cause: поле опциональное, но без проверки на null
Solution: добавлена проверка перед обращением
Prevention: всегда проверять опциональные поля из внешних источников
Tags: #null-check #optional-field
```

Перед каждой новой задачей AI читает патчи. Ошибки не повторяются. Это та же идея, что описана в практике `05-memory.md` (слой 3: Карточки ловушек), но здесь она автоматизирована.

## Spec-driven подход

AI Factory работает по спецификации, а не «вслепую»:
1. Сначала исследование (`/aif-explore`) — какие варианты есть
2. Потом план (`/aif-plan`) — конкретные задачи с зависимостями
3. Только потом реализация (`/aif-implement`) — строго по плану

Это предотвращает ситуацию, когда AI бросается писать код, не разобравшись в задаче.

## Адаптация под вашу область

Принцип конвейера навыков применим не только к разработке:
- `/research "Тема"` → план исследования + сбор данных
- `/report` → оформление результатов по шаблону
- `/check` → проверка по чеклисту

Разбить работу на этапы и автоматизировать каждый через навык — работает в любой сфере.

Подробнее об установке и настройке: `recipes/install-ai-factory.md`

## Superpowers + AI Factory: как они работают вместе

### Дополняют, не конфликтуют

Это инструменты разного уровня, которые усиливают друг друга:

- **Superpowers** работает на уровне **поведения AI** — меняет *как* Claude Code думает: мозговой штурм перед действием, поиск корневой причины при ошибке, проверка перед «готово». Не создаёт файлов — меняет процесс мышления.

- **AI Factory** работает на уровне **управления задачами** — организует *что* делается: создаёт планы, ведёт чеклисты, коммитит на чекпоинтах, сохраняет патчи ошибок. Создаёт артефакты (PLAN.md, патчи, спецификации).

### Как они взаимодействуют на практике

| Этап | Superpowers | AI Factory |
|-|-|-|
| Новая задача | Запускает мозговой штурм, уточняет требования | `/aif-explore` исследует варианты |
| Планирование | Требует план перед реализацией | `/aif-plan` создаёт PLAN.md с чеклистом |
| Реализация | Следит за качеством каждого шага | `/aif-implement` выполняет по плану с коммитами |
| Ошибка | Ищет корневую причину, не пробует наугад | `/aif-fix` создаёт патч для будущего обучения |
| Завершение | Требует реальную проверку перед «готово» | Патчи и навыки обновляются автоматически |

### Аналогия

**Superpowers** — это характер специалиста (дисциплинированный, вдумчивый, проверяет за собой).
**AI Factory** — это его рабочий стол (план-доска, чеклисты, журнал ошибок, инструменты).

Можно использовать по отдельности:
- Только Superpowers → AI работает дисциплинированно, но без формальных артефактов
- Только AI Factory → AI ведёт планы и патчи, но может пропустить мозговой штурм

Вместе — максимальный эффект: дисциплина мышления + структура работы.

### Порядок установки

1. Сначала **Superpowers** — меняет поведение AI сразу, работает в любой сфере
2. Потом **AI Factory** — когда начнёте работать с конкретными задачами, особенно в разработке
