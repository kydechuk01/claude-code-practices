# CLAUDE.md — инструкция для AI

## Суть
Файл `CLAUDE.md` в корне проекта — автоматически загружается в каждую сессию Claude Code. Без него AI не знает стек, конвенции и известные ловушки проекта.

## Что включать
- **Стек** — технологии, версии, особенности сборки
- **Конвенции** — код-стайл, именование, структура папок
- **Известные ловушки** — задокументированные проблемы (на которые уже потрачено время)
- **Маппинг агентов** — какой агент для какой задачи

## Размеры в реальных проектах
- Небольшой проект: ~100 строк (стек + правила)
- Средний проект: ~200 строк (стек + правила + маппинг агентов + ловушки)
- Сложный проект (инфраструктура, сетевое оборудование): ~400 строк (полная конфигурация)

## Известные ловушки — главная ценность
Каждая ловушка — проблема, на которую уже потрачено время. Формат:
- Идентификатор (G-01, G-02...)
- Краткое описание проблемы
- Решение

Примеры (зависят от вашей области):
- Кодировка файлов из внешних систем не совпадает с ожидаемой → конвертировать при импорте
- Сервис кэширует DNS после перезапуска зависимости → настроить динамический резолвинг
- Политики безопасности блокируют inline-ресурсы → настроить исключения
- Выгрузка данных содержит записи за предыдущий период → фильтровать по дате документа

## Принцип
В CLAUDE.md — только то, что нужно в каждой сессии. Всё остальное — в reference files.
