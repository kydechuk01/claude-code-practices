# CLAUDE.md — инструкция для AI

## Суть
Файл `CLAUDE.md` в корне проекта — автоматически загружается в каждую сессию Claude Code. Без него AI не знает стек, конвенции и известные ловушки проекта.

## Что включать
- **Стек** — технологии, версии, особенности сборки
- **Конвенции** — код-стайл, именование, структура папок
- **Известные ловушки** — задокументированные проблемы (на которые уже потрачено время)
- **Маппинг агентов** — какой агент для какой задачи

## Размеры в реальных проектах
- Небольшой проект: ~100 строк (стек + правила)
- Средний проект: ~200 строк (стек + правила + маппинг агентов + ловушки)
- Сложный проект (инфраструктура, сетевое оборудование): ~400 строк (полная конфигурация)

## Известные ловушки — главная ценность

Каждая ловушка — проблема, на которую уже потрачено время. Два уровня хранения:

**Реестр в CLAUDE.md** — компактная таблица (до 20 строк), загружается автоматически:

| # | Ловушка | Решение | Дата |
|-|-|-|-|
| G-01 | Кодировка файлов из внешних систем | Конвертировать при импорте | 2026-01-15 |
| G-02 | Кэш DNS после перезапуска зависимости | Динамический резолвинг | 2026-02-03 |

**Карточки в `_docs/gotchas/`** — подробное описание каждой ловушки (причина, предотвращение). Формат:

```
# G-01: [заголовок]
Дата | Критичность | Область
## Проблема → Причина → Решение → Предотвращение
```

Протокол: проблема заняла >10 мин или повторилась → карточка → строка в реестр → если следует правило → эскалация в «Обязательные правила».

## Принцип
В CLAUDE.md — только то, что нужно в каждой сессии. Всё остальное — в reference files.

## Метаобучение — инструкция растёт из практики

CLAUDE.md — не статичный документ, написанный на старте. Это живой артефакт, который растёт из опыта: каждая ловушка, каждый удачный паттерн, каждое решение — становятся частью инструкции.

Принцип: **работай → находи паттерн → записывай → автоматизируй**. Через месяц активной работы CLAUDE.md превращается в концентрированную базу знаний, которая делает AI эффективнее с каждой неделей.
