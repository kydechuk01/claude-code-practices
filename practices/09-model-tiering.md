# Тиринг моделей — оптимизация бюджета

## Суть
Не все задачи требуют самой мощной (и дорогой) модели. Указание модели в описании агента позволяет контролировать расходы.

## Распределение

| Модель | Для чего | Примеры агентов |
|--------|----------|-----------------|
| opus   | Сложный анализ, архитектура, ревью | systems-analyst, art-director, code-reviewer |
| sonnet | Процедурные задачи, деплой | deploy-operator, реализация по плану |
| haiku  | Мониторинг, диагностика, простые проверки | log-diagnostics, проверки статуса |

## Влияние на бюджет
В сессии с активным использованием субагентов расход растёт кратно. Тиринг позволяет держать бюджет под контролем при интенсивной мультиагентной работе.

## Подписки
- **Pro** ($20/мес) — достаточен для базовой работы с одним агентом. При мультиагентной нагрузке лимит исчерпывается быстро (за 10-15 минут интенсивной сессии)
- **Max 5** ($100/мес) — оптимум для одного активного проекта с агентами
- **Max 20** ($200/мес) — для параллельной работы над 2-3 проектами

Один аккаунт Max 5 ($100) удобнее нескольких Pro: единая memory, единые настройки, не нужно переключаться между аккаунтами.
