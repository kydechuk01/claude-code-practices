# Claude Code Practices

**Автор:** Александр Климок, 2026
**Язык:** русский / Language: Russian

Настройте Claude Code один раз — и он будет помнить ваш контекст, проверять факты, работать по вашим правилам. И не терять рабочую нить: каждая новая сессия продолжает с того места, где предыдущая остановилась.

**9 практик** с пояснениями, **17 готовых рецептов** настройки. Интерактивный чеклист, который адаптируется под любую вашу сферу — разработка ПО, аналитика, управление проектами, юриспруденция, медицина.

## Что нового (v1.7)

Релиз посвящён **передаче контекста между сессиями** — самой больной точке длинных проектов на Claude Code. Раньше была одна строка-указатель «где я остановился» (рецепт `track-progress`); теперь добавлена полноценная сеансовая эстафета — для длинных пауз, передачи коллеге и фазовых переходов.

- **Новый рецепт «Сеансовая эстафета»** (session handoff) — самодостаточный документ, по которому новая чистая сессия за 10 минут восстанавливает рабочий контекст. Метафора прямая: эстафетная палочка, без обрыва. Главный принцип — эстафетный файл это не отчёт о сделанном, а исполнимая программа холодного старта новой сессии.
- **Рецепт построен на анализе трёх независимых рабочих практик** из разных доменов — разработка, юридическая практика, бухгалтерия с аналитикой. Все три проекта пришли к одной архитектуре самостоятельно: гарантированная доставка ядра + восемь обязательных разделов + расширения по профилю проекта. Это снимает selection bias одного теста.
- **Новое правило 7 в `behavior-rules.md` — «Надёжность доставки критичных инструкций».** Эмпирически подтверждённое: одной записи в CLAUDE.md недостаточно для критичных триггеров. Надёжность даёт SessionStart hook, инжектирующий контекст в system-reminder до первого ответа модели. Правило оформляет границы применимости CLAUDE.md.
- **Invariant-pointer LATEST** вместо поиска по дате — `docs/SESSION-HANDOFF-LATEST.md` как стабильный указатель. Hook не нужно править при каждом новом handoff. Раздел про подводные камни на Windows и в монорепе — отдельной секцией в рецепте.
- **Чёткая граница между `track-progress.md` и `session-handoff.md`** с явным алгоритмом выбора: до 2 часов работы → закладка в книге, дольше или фазовый переход → досье по делу.

Ранее в v1.6: параллельные агенты с watchdog. В v1.5: Anti-Hallucination протокол, унификация журнала ловушек, профессиональная автономия агентов, агенты-персоны пользователей. В v1.4: непрерывность контекста. В v1.3: активация навыков. В v1.2: MCP-серверы. В v1.1: RTK-оптимизация.

## Идея

Настройка Claude Code — через разговор с ним же.

**Перед вами интерактивный AI-инсталлятор, написанный полностью на человеческом языке!** Никакого кода, никаких скриптов и конфигов. Возможно, это первый в мире случай, когда README — и есть программа.

Запуск — файл START.md. Вы говорите Claude Code «прочитай его и проведи меня по настройке» — и дальше он сам читает инструкции, спрашивает о вашей работе, адаптирует примеры под вашу сферу и контекст, настраивает окружение шаг за шагом. Вы просто отвечаете на вопросы и одобряете результат. В любой момент можно прервать или отменить инсталляцию и потом к ней вернуться, ИИ запомнит, где вы остановились и какие настройки уже сделаны.

Весь пакет также написан при помощи Claude Code, с применением большинства описанных здесь практик.


## На чём основано

Ежедневная работа с Claude Code на проектах, у которых нет ничего общего между собой: веб-платформы с полным циклом разработки, координированная настройка сетевой инфраструктуры, аудит веб-сайтов, сбор финансовой и бизнес-аналитики, тестирование корпоративной BPM-системы, юридическая практика, бухгалтерия. Разные домены сошлись на одних и тех же приёмах — это и есть критерий, по которому практика попадает в пакет.

Чтобы прийти к этому набору, пришлось пересмотреть десятки YouTube-роликов и разобраться:

- как вести мега-сессии на десятки дней и не терять управление контекстом
- как передавать состояние проекта между сессиями без обрыва нити
- как запускать несколько AI-агентов на одну задачу с разными ролями
- как строить автономную работу без участия оператора
- как организовывать память проекта — так, чтобы он сам за собой следил

Что-то заработало с первого раза, что-то приходилось переделывать. В итоге здесь отобрано то, что реально работает на нескольких проектах одновременно. Работа над пакетом продолжается.


## Что даёт

| Без практик | С практиками |
|---|---|
| AI не знает контекст — объясняете заново каждый раз | CLAUDE.md хранит контекст, AI работает сразу |
| AI иногда додумывает то, чего нет | Правила поведения требуют проверять факты |
| Один промпт — один запрос | Агенты-специалисты решают типовые задачи |
| Контекст теряется между сессиями | Память сохраняет решения и факты |
| После паузы — снова восстанавливать «где остановились» | Сеансовая эстафета: новая сессия читает ядро за минуту и продолжает |
| Каждая задача — с нуля | AI Factory: конвейер от исследования до результата |

Настройка фундамента — **30 минут**. Полный чеклист — **1-2 часа**.

## Быстрый старт

> **Требования:**
> - [Claude Code](https://docs.anthropic.com/en/docs/claude-code) должен быть установлен. Если нет — [инструкция](https://docs.anthropic.com/en/docs/claude-code/getting-started)
> - **Рекомендуемая модель: Sonnet или выше.** На модели Haiku настройка может работать менее точно

### Шаг 1. Скачайте и распакуйте

Нажмите зелёную кнопку **Code** вверху этой страницы → **Download ZIP**.

Распакуйте архив **куда удобно** — на рабочий стол, в папку «Документы», на сетевой диск, куда угодно. Пакет — это справочник, а не программа. Ему не нужно быть в определённом месте.

> **Примечание:** при распаковке ZIP с GitHub создаётся папка с суффиксом `-main` (например, `claude-code-practices-main`). Это нормально — внутри неё всё содержимое пакета.

### Шаг 2. Откройте Claude Code в вашей рабочей папке

Откройте Claude Code в той папке, где вы обычно работаете — ваш проект, рабочие документы, данные. Это важно: настройка будет выполнена именно для этой папки.

> **Если вы уже запускаете Claude Code** из определённой папки — запустите как обычно и переходите к шагу 3.

> **Если вы ещё не запускали Claude Code самостоятельно** — попросите коллегу, который вам его показал, помочь с первым запуском в вашей рабочей папке. Это занимает 1 минуту. После первого раза вы будете запускать так же.

Для тех, кто знаком с терминалом — перейдите в папку проекта и запустите `claude`:

**Windows (PowerShell):**
```
cd "D:\Работа\Мой проект"
claude
```

**macOS / Linux:**
```
cd ~/my-project
claude
```

### Шаг 3. Скажите Claude Code одну фразу

Скопируйте путь к файлу START.md из распакованной папки и скажите Claude Code:

```
Прочитай файл [путь к START.md] и проведи меня по настройке.
```

**Как узнать путь к файлу:**
- **Windows:** откройте папку пакета в Проводнике, найдите файл START.md, кликните правой кнопкой → «Копировать как путь». Вставьте в Claude Code
- **macOS:** перетащите файл START.md из Finder в окно терминала — путь вставится автоматически

**Примеры (зависят от того, куда вы распаковали):**

```
Прочитай файл D:\Загрузки\claude-code-practices-main\START.md и проведи меня по настройке.
```

```
Прочитай файл C:\Users\Марина\Desktop\claude-code-practices-main\START.md и проведи меня по настройке.
```

### Что произойдёт дальше

Claude Code прочитает инструкцию и начнёт интерактивную настройку:
- Спросит, чем вы занимаетесь (разработка, аналитика, документы...)
- Адаптирует все примеры под вашу сферу
- На каждом шаге объяснит, зачем это нужно
- Вы решаете, что настроить сейчас, а что отложить на потом

## Что создаёт пакет

Сам пакет остаётся справочником — его содержимое **не изменяется**. Все настройки создаются в других местах:

**В вашей рабочей папке** (откуда вы запустили Claude Code):
- `CLAUDE.md` — файл с контекстом вашей работы. Claude Code читает его автоматически при каждом запуске
- `docs/SESSION-HANDOFF-*.md` — эстафетные файлы, если включена практика сеансовой эстафеты (создаются в конце дня или при передаче проекта)

**В настройках Claude Code** (создаются автоматически, вам не нужно знать точный путь):
- Агенты — специалисты под ваши задачи
- Навыки — автоматические процедуры (вызываются командами типа `/silent-mode`)
- Память — контекст, который сохраняется между запусками Claude Code
- SessionStart hook — гарантирует, что критичный контекст загружается при старте каждой сессии

> После настройки пакет можно удалить — все созданные настройки продолжат работать без него.

## Экономия токенов — RTK

[RTK](https://github.com/rtk-ai/rtk) — CLI-прокси, который сжимает вывод консольных команд перед попаданием в контекст Claude Code. Один бинарник, без зависимостей, задержка <10 мс.

Реальные результаты на React-проекте (5.8 МБ, 243 JSX-файла):

| Операция | Без RTK | С RTK | Экономия |
|-|-|-|-|
| Рекурсивный листинг каталога | 39.8 КБ | 0.2 КБ | **99.5%** |
| Поиск файлов (243 .jsx) | 79.1 КБ | 1.8 КБ | **97.7%** |
| Поиск по коду (`grep`) | 64.3 КБ | 0.6 КБ | **99.0%** |
| Чтение файла (aggressive) | 5.4 КБ | 1.2 КБ | **77.8%** |

Подробнее: `recipes/optimize-tokens.md`

## Ключевые принципы

1. **CLAUDE.md — главный актив.** Минута на качество экономит часы.
2. **Правила > надежды.** Пропишите поведение AI явно.
3. **Агенты > промпты.** Настроенный агент лучше длинного промпта.
4. **Память > повторение.** AI помнит контекст, а не вы пересказываете.
5. **Ответственность > угодливость.** AI не должен молча делать глупость.
6. **Эстафета > перезапуск.** Сессия передаёт состояние следующей, а не уходит в пустоту.
7. **Доставка > надежда.** Критичные триггеры доставляются механизмом, а не одной строкой в CLAUDE.md.

## Несколько проектов

Пакет скачивается **один раз**. Для каждого проекта запускайте START.md отдельно — откройте Claude Code в папке нужного проекта и передайте путь к START.md. Claude Code создаст отдельные настройки для каждого проекта.

## Обновление

1. Скачайте новый ZIP с этой страницы
2. Распакуйте в новую папку (старую можно удалить — ваши настройки хранятся отдельно)
3. Запустите START.md в каждом проекте, указав путь к новой папке

Claude Code определит, что уже настроено, и предложит только новые практики. Существующие настройки не будут изменены без вашего согласия.

## Удаление

**Удалить пакет:** просто удалите папку с пакетом (через Проводник, Finder или файловый менеджер). Это безопасно — ваши настройки продолжат работать.

**Удалить созданные настройки (опционально):** откройте Claude Code и скажите: «Удали из CLAUDE.md секции, которые были добавлены пакетом claude-code-practices. Также удали агентов, навыки и память, которые были созданы при настройке этого пакета. Мои собственные настройки не трогай». Claude Code найдёт блоки, помеченные меткой пакета, и удалит только их.
