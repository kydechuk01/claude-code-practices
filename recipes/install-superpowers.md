# Как установить и использовать плагин Superpowers

## Что это

Superpowers — это набор из 14 навыков (skills), которые превращают Claude Code из «помощника, который отвечает на вопросы» в **систему с дисциплиной работы**. Плагин автоматически подключается к каждой сессии и срабатывает, когда задача попадает под один из навыков.

Без Superpowers вы сами должны помнить: «перед кодом — спланируй», «перед фиксом — найди причину», «перед тем как сказать готово — проверь». С Superpowers это зашито в поведение Claude Code.

## Установка

Выполните в обычном терминале (не внутри Claude Code).

**Шаг 1: Подключите маркетплейс Superpowers** (один раз):

```
claude plugins marketplace add github:obra/superpowers-marketplace
```

**Шаг 2: Установите плагин:**

```
claude plugins install superpowers@superpowers-marketplace
```

> **Если шаг 1 не работает** (команда `marketplace add` недоступна в вашей версии), попробуйте напрямую:
> ```
> claude plugins install superpowers@claude-plugins-official
> ```

После установки перезапустите Claude Code — навыки активируются автоматически.

**Проверка:** выполните `claude plugins list` — в списке должен быть `superpowers`.

### Если установка не удалась

Плагин Superpowers может быть недоступен в вашем маркетплейсе (зависит от версии Claude Code и региона). В этом случае — добавьте базовые правила дисциплины прямо в CLAUDE.md. Это не полная замена плагина (14 навыков невозможно заменить 5 правилами), но покрывает основное: «думай перед действием» и «проверяй перед завершением».

Используйте правила из `recipes/behavior-rules.md` и дополните блоком:

```markdown
## Дисциплина работы
- Перед новой задачей: уточни, предложи 2-3 варианта, дождись выбора
- Перед большой работой: разбей на шаги, покажи план, дождись подтверждения
- При ошибке: прочитай ошибку целиком, найди причину, только потом чини
- Перед «готово»: выполни реальную проверку, не говори «должно работать»
```

Когда `/install-plugin` станет доступен — установите плагин и удалите этот блок из CLAUDE.md.

## Что даёт плагин

### Перед любым действием — думай

**Brainstorming (мозговой штурм).** Прежде чем что-то делать, Claude Code проведёт вас через уточнение задачи: что именно нужно, какие варианты есть, какие ограничения. Результат — проработанная спецификация, а не импровизация на ходу.

Когда срабатывает: любая творческая задача — новая функция, новый документ, изменение поведения.

Что делает:
- Задаёт уточняющие вопросы (Сократовский диалог)
- Предлагает 2-3 варианта подхода с плюсами и минусами
- Сохраняет итоговую спецификацию в файл для дальнейшей работы

### Перед работой — спланируй

**Writing Plans (написание плана).** После утверждения спецификации Claude Code разбивает работу на маленькие задачи (по 2-5 минут каждая). Каждая задача: что делать, какие файлы затронуты, как проверить результат.

Зачем это нужно:
- Без плана Claude Code может пойти не в ту сторону, и вы потеряете время
- План — это контракт: вы видите, что будет делаться, и можете скорректировать до начала работы
- При длинной сессии или Silent Mode — план не даёт сбиться с пути

### При ошибках — ищи причину, а не чини симптомы

**Systematic Debugging (систематическая отладка).** Когда что-то ломается, Claude Code НЕ будет сразу предлагать «попробуй заменить X на Y». Вместо этого:

1. Прочитать ошибку внимательно
2. Воспроизвести стабильно
3. Найти корневую причину
4. Сформулировать одну гипотезу
5. Проверить минимальным изменением
6. Только потом чинить

Ключевое правило: **если 3 попытки починить не помогли — проблема архитектурная**, не пытаться чинить дальше, а пересмотреть подход.

### Перед тем как сказать «готово» — докажи

**Verification Before Completion (проверка перед завершением).** Claude Code не скажет «готово», пока не выполнит реальную проверку: запустит команду, прочитает вывод, убедится, что всё работает. Никаких «должно работать» и «вроде всё ок».

### Параллельная работа

**Dispatching Parallel Agents (параллельные агенты).** Если задачу можно разделить на независимые части — Claude Code запустит несколько агентов параллельно. Три задачи решаются за время одной.

**Subagent-Driven Development (разработка через субагентов).** При выполнении плана каждую задачу делает отдельный агент, после чего результат проходит двухстадийное ревью: сначала на соответствие спецификации, потом на качество.

### Организация работы

**Using Git Worktrees (рабочие копии).** Для изоляции: новая задача делается в отдельной копии репозитория, не мешая основной ветке.

**Finishing a Development Branch (завершение ветки).** Когда работа выполнена, Claude Code предложит 4 варианта: слить локально, создать PR, оставить как есть, или отменить. Структурированный выбор вместо хаоса.

## Полный рабочий цикл с Superpowers

```
1. Мозговой штурм     → уточнение задачи, спецификация
2. Написание плана    → разбивка на задачи по 2-5 минут
3. Рабочая копия      → изоляция от основного кода
4. Выполнение плана   → по задачам, с ревью после каждой
5. Отладка            → систематическая, по корневой причине
6. Проверка           → реальная, не «вроде работает»
7. Завершение         → merge / PR / оставить
```

Каждый шаг срабатывает автоматически — вам не нужно помнить о них.

## Для кого это полезно

- **Для разработчиков** — полный цикл: план → код → тесты → ревью → деплой
- **Для аналитиков** — brainstorming для уточнения задачи, планирование этапов анализа, верификация результатов
- **Для любых задач** — дисциплина «думай → планируй → делай → проверяй» полезна везде

## Важно понимать

Superpowers — это не магия, а **дисциплина, зашитая в инструмент**. Плагин не делает Claude Code умнее, но заставляет его:
- Думать перед действием
- Планировать перед реализацией
- Проверять перед завершением
- Не угадывать причину ошибки, а искать

Это именно те привычки, которые отличают эффективную работу от хаотичной.
