# Мультиагентная коллегиальная работа

## Идея

Один специалист не может быть одновременно бухгалтером, юристом и аудитором. То же с AI: один агент-генералист даёт средний результат. Несколько агентов с разными ролями, работающих над одной задачей — дают результат, который каждый по отдельности не выдал бы.

Коллегиальная работа — это когда вы поручаете задачу нескольким агентам, каждый анализирует со своей стороны, потом они перекрёстно проверяют друг друга, и агент-оркестратор собирает итоговый результат.

## Когда это оправдано

- Задача требует экспертизы в нескольких областях одновременно
- Цена ошибки высока — нужна перекрёстная проверка
- Задача объёмная и части можно обработать параллельно
- Нужно рассмотреть проблему с разных точек зрения

Когда НЕ нужно: простые однозначные задачи, где один агент справится за 5 минут.

## Три паттерна

### Паттерн 1: Параллельный анализ + синтез

Несколько агентов работают над одной задачей независимо, каждый в своей роли. Потом оркестратор сводит результаты.

```
         ┌─ Агент-роль-1 ─┐
Задача ──┤  Агент-роль-2  ├── Оркестратор ── Итог
         └─ Агент-роль-3 ─┘
```

**Фаза 1 — Параллельная работа:** каждый агент получает одну и ту же задачу, но работает в рамках своей роли. Результат — структурированный отчёт от каждого.

**Фаза 2 — Синтез:** агент-оркестратор читает все отчёты, находит пересечения, противоречия и пробелы. Формирует итоговый документ.

**Когда использовать:** анализ данных, подготовка отчётов, оценка рисков — любая задача, где важен взгляд с разных сторон.

### Паттерн 2: Перекрёстная проверка

Каждый агент не только делает свою часть, но и проверяет результаты других. Это ловит ошибки, которые автор не замечает в своей работе.

```
Агент-1 ──делает──► Результат-1 ──проверяет──► Агент-2
Агент-2 ──делает──► Результат-2 ──проверяет──► Агент-3
Агент-3 ──делает──► Результат-3 ──проверяет──► Агент-1
```

**Фаза 1 — Работа:** каждый агент выполняет свою часть задачи.
**Фаза 2 — Ревью:** каждый получает результат другого агента и проверяет: корректность, полнота, логика.
**Фаза 3 — Исправления:** авторы учитывают замечания, оркестратор собирает финал.

**Когда использовать:** когда важна безошибочность — финансовые расчёты, юридические документы, медицинские заключения.

### Паттерн 3: Волновая обработка

Задача проходит через несколько волн, где каждая волна — это группа агентов с определённой функцией. Результат одной волны — входные данные для следующей.

```
Волна 1: Сбор        → Агенты-исследователи (параллельно)
Волна 2: Анализ       → Агенты-аналитики (параллельно)
Волна 3: Синтез       → Агент-оркестратор
Волна 4: Оформление   → Агент-редактор
```

**Когда использовать:** сложные проекты с чёткими этапами — исследования, подготовка комплексных документов, аудиты.

## Роль оркестратора

Оркестратор — это агент, который координирует работу остальных. Его задачи:
- Распределить работу между агентами
- Собрать результаты
- Найти противоречия и потребовать обоснования от каждой стороны
- Принять решение или вынести спорные вопросы на пользователя
- Оформить итоговый документ

Оркестратор должен быть на самой мощной модели (opus), потому что его работа — самая сложная: синтез, сравнение, принятие решений.

## Примеры ролей для разных сфер

| Сфера | Роль 1 | Роль 2 | Роль 3 | Оркестратор |
|-|-|-|-|-|
| Финансы | Бухгалтер (сводка операций) | Финаналитик (P&L, тренды) | Налоговик (риски, оптимизация) | Финдиректор (синтез) |
| Медицина | Терапевт (анамнез, общая картина) | Узкий специалист (профильный анализ) | Фармаколог (взаимодействие препаратов) | Консилиум (итоговое заключение) |
| Юриспруденция | Юрист по договорам (риски контракта) | Налоговый юрист (налоговые последствия) | Комплаенс (регуляторные требования) | Старший партнёр (итоговое заключение) |
| Исследования | Сборщик данных (факты, источники) | Аналитик (закономерности, выводы) | Критик (слабые места, контраргументы) | Научный руководитель (синтез) |
| Разработка | QA-инженер (тесты, edge-cases) | Разработчик (архитектура, реализация) | UX-дизайнер (пользовательский опыт) | Тех-лид (итоговое решение) |
| Контент | Автор (текст) | Фактчекер (проверка данных) | Редактор (стиль, структура) | Главред (финальная версия) |

## Как запустить

Опишите задачу Claude Code и попросите организовать коллегиальную работу:

> Мне нужно [задача]. Организуй коллегиальную работу: запусти нескольких агентов с разными ролями, пусть каждый проанализирует задачу со своей стороны, потом сверь результаты и собери итоговый документ.

Claude Code сам определит подходящие роли, организует фазы и соберёт результат.

## Готовые шаблоны промптов

Если вы хотите больше контроля над процессом, можете использовать готовый шаблон. Попросите Claude Code:

> Подготовь шаблон промпта для коллегиальной работы под мою задачу: [опишите задачу]. Включи: роли агентов, фазы работы, формат вывода каждого агента, инструкцию для оркестратора.

Claude Code сгенерирует шаблон, адаптированный под вашу задачу и сферу. Шаблон можно сохранить как навык и использовать повторно.

### Пример шаблона (финансовый анализ):

```
Организуй коллегиальную работу по задаче: квартальный отчёт Q1 2026.

Фаза 1 — Параллельный анализ:
- Агент «Бухгалтер»: сводка операций, баланс, движение средств. Каждая цифра — с источником.
- Агент «Аналитик»: P&L, маржинальность, тренды vs предыдущий квартал.
- Агент «Налоговик»: налоговая нагрузка, риски, рекомендации по оптимизации.

Фаза 2 — Перекрёстная проверка:
Каждый агент получает результаты двух других. Задача: найти противоречия в цифрах, пробелы в анализе, неучтённые факторы.

Фаза 3 — Синтез (оркестратор «Финдиректор»):
- Согласованные выводы (с подтверждением от всех трёх)
- Разногласия (позиция каждой стороны с обоснованием)
- Итоговые рекомендации
- Формат: структурированный отчёт с таблицами
```

## Контроль бюджета

При мультиагентной работе важен тиринг моделей:
- **Оркестратор** — opus (сложный синтез, принятие решений)
- **Аналитические роли** — opus или sonnet (зависит от сложности)
- **Сбор данных, оформление** — sonnet или haiku (процедурные задачи)

Без тиринга сессия с 5+ агентами может стоить в 3-5 раз дороже, чем нужно.

## Результат

Коллегиальная работа даёт:
- **Полноту** — каждый аспект задачи покрыт специалистом
- **Надёжность** — перекрёстная проверка ловит ошибки
- **Скорость** — параллельная работа быстрее последовательной
- **Качество синтеза** — оркестратор видит картину целиком, а не отдельные фрагменты
