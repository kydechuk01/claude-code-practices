# Инвентаризация проекта — поддержание порядка

## Проблема

Проект живёт: появляются новые файлы, удаляются старые, меняются решения. CLAUDE.md устаревает: ссылки ведут в никуда, упомянутые файлы не существуют, новые важные документы не упомянуты. Reference files расходятся с реальностью. AI работает с устаревшим контекстом — и выдаёт устаревшие решения.

## Решение: регулярная инвентаризация

Периодически (раз в неделю при активной работе, раз в месяц при спокойной) запускайте полную проверку проекта. Это занимает 5-10 минут, а экономит часы на будущих сессиях.

## Промпт для инвентаризации

Скопируйте и вставьте в Claude Code:

```
Проведи полную инвентаризацию проекта:

1. Аудит файлов — найди все документы, данные, конфиги в рабочей директории
2. Проверка ссылок — все ссылки в CLAUDE.md и MEMORY.md ведут на существующие файлы?
3. Обратная проверка — есть ли важные файлы, которые НЕ упомянуты в CLAUDE.md?
4. Консистентность — нет ли противоречий между документами?
5. Актуальность ловушек — проверить каждую строку в реестре CLAUDE.md: актуальна ли проблема? Устаревшие — пометить в карточке (`_docs/gotchas/`) как УСТАРЕЛА, убрать из реестра. Карточки не удалять.
6. Актуальность reference files — устарели ли справочные документы?
7. Оптимизация CLAUDE.md — обнови файл: добавь новое, убери устаревшее,
   актуализируй ссылки. Цель: после любого перерыва сессия начинается
   с максимально актуальным контекстом.

Результат: отчёт с найденными проблемами + обновлённый CLAUDE.md.
```

## Что это даёт

- **Битые ссылки** — CLAUDE.md ссылается на `ref-architecture.md`, а файл переименован в `architecture-v2.md`. AI не находит его и работает без контекста.
- **Потерянные файлы** — за неделю появились новые спецификации, но CLAUDE.md о них не знает. AI не использует их при работе.
- **Устаревшие ловушки** — ловушка G-03 описывает проблему, которую уже починили. AI тратит время на обход несуществующей проблемы.
- **Противоречия** — в одном файле написано «статусы заявки: 5», в другом — «7 статусов». AI путается.

## Как внедрить

При настройке через START.md Claude Code добавит в ваш CLAUDE.md правило:

```markdown
## Обслуживание проекта

### Инвентаризация
Раз в неделю (при активной работе) или после крупных изменений — запросите:
«Проведи инвентаризацию проекта» — полный аудит файлов, ссылок,
консистентности и актуализация CLAUDE.md.
```

Этого достаточно: Claude Code увидит правило и при необходимости напомнит о проверке. Либо вы сами запускаете промпт, когда чувствуете, что «что-то устарело».

## Когда запускать

- После крупного этапа работы (завершена фаза, сдан документ)
- Перед началом нового этапа (чтобы стартовать с чистым контекстом)
- Если AI начал давать странные ответы (возможно, работает с устаревшими данными)
- Раз в неделю как профилактика
