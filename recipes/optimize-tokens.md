# Оптимизация расхода токенов — RTK

## Что это

RTK (Rust Token Killer) — CLI-прокси, который фильтрует и сжимает вывод консольных команд прежде, чем они попадут в контекст Claude Code. Один бинарник на Rust, без зависимостей, задержка менее 10 мс.

Без RTK Claude Code получает полный сырой вывод каждой команды — тысячи строк, из которых полезны десятки. С RTK — только суть: ошибки, изменённые файлы, структура, результаты. Всё лишнее отсекается автоматически.

## Зачем

Каждый токен — это деньги (на платных планах) и место в контекстном окне. Чем больше мусора в контексте — тем хуже Claude Code понимает задачу и тем быстрее упирается в лимит. RTK убирает шум, оставляя только сигнал.

## Реальные результаты тестирования

Тест на React-проекте (5.8 МБ исходников, 243 JSX-файла):

| Операция | Без RTK | С RTK | Экономия |
|-|-|-|-|
| `ls -R` (рекурсивный листинг каталога) | 39.8 КБ | 0.2 КБ | **99.5%** |
| `find *.jsx` (поиск 243 файлов) | 79.1 КБ | 1.8 КБ | **97.7%** |
| `grep "import.*React"` (поиск по проекту) | 64.3 КБ | 0.6 КБ | **99.0%** |
| `read` файла (aggressive mode) | 5.4 КБ | 1.2 КБ | **77.8%** |
| `git status` | 838 байт | 215 байт | **74%** |

Общая экономия за тестовую сессию: **~50% токенов**. На длинных сессиях с частым поиском и листингом — до 80-90%.

## Как работает

```
Без RTK:   Claude Code → команда → shell → полный вывод (тысячи строк) → контекст
С RTK:     Claude Code → команда → RTK → shell → фильтрация → суть (десятки строк) → контекст
```

Четыре стратегии сжатия:
1. **Фильтрация** — убирает шум: пустые строки, комментарии, бойлерплейт
2. **Группировка** — файлы по папкам, ошибки по типам
3. **Обрезка** — оставляет релевантное, убирает повторы
4. **Дедупликация** — одинаковые строки → одна строка + счётчик

## Установка

### macOS (Homebrew)
```bash
brew install rtk
```

### Linux
```bash
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh
```

### Windows
Скачайте бинарник с [releases](https://github.com/rtk-ai/rtk/releases) (файл `rtk-x86_64-pc-windows-msvc.zip`), распакуйте в удобную папку и добавьте её в PATH.

### Проверка
```bash
rtk --version
```

## Подключение к Claude Code

### macOS / Linux — автоматический hook
```bash
rtk init -g
```
Прописывает hook в `~/.claude/settings.json`. После этого Claude Code автоматически подменяет команды: `git status` → `rtk git status`. Claude Code не знает о подмене — просто получает сжатый вывод.

### Windows — инструкции в CLAUDE.md
```bash
rtk init -g
```
На Windows hook не поддерживается. RTK автоматически переключается в режим `--claude-md`: добавляет инструкции в `~/.claude/CLAUDE.md`, которые говорят Claude Code самому использовать `rtk` перед командами.

### Перезапустите Claude Code после установки.

## Основные команды

### Файлы и поиск
```bash
rtk ls .                        # Компактный листинг каталога
rtk find "*.jsx" .              # Поиск файлов, группировка по папкам
rtk grep "pattern" .            # Поиск по содержимому, группировка
rtk read file.js                # Чтение файла с фильтрацией
rtk read file.js -l aggressive  # Только сигнатуры (без тел функций)
```

### Git
```bash
rtk git status                  # Компактный статус
rtk git log -n 10               # Однострочный лог
rtk git diff                    # Сжатый diff
rtk git add/commit/push         # Минимальный вывод: «ok abc1234»
```

### Тесты (только ошибки)
```bash
rtk test cargo test             # Rust — только падения
rtk vitest run                  # Vitest — только падения
rtk pytest                      # Python — только падения
rtk go test                     # Go — только падения
```

### Сборка и линтинг
```bash
rtk tsc                         # TypeScript — ошибки по файлам
rtk lint                        # ESLint — группировка по правилам
rtk cargo build                 # Cargo — компактный вывод
```

## Проверка экономии

```bash
rtk gain                        # Статистика сэкономленных токенов
rtk gain --history              # История команд с экономией
```

## Ограничения

- **Только Bash-команды.** Встроенные инструменты Claude Code (Read, Grep, Glob) не проходят через RTK — они работают напрямую. Для экономии на этих операциях используйте shell-аналоги: `rtk read` вместо Read, `rtk grep` вместо Grep
- **Windows:** hook-режим не поддерживается, работает через инструкции в CLAUDE.md
- **Markdown-файлы:** RTK не сжимает текстовые файлы (нечего фильтровать). Максимальная экономия — на коде, логах, выводе команд
