# Как создавать кастомных агентов

## Что такое агент

Агент — md-файл в `~/.claude/agents/`, описывающий роль, экспертизу и правила поведения. Claude Code вызывает его через Agent tool, и AI работает в рамках заданной роли, а не как универсальный генералист.

## Когда создавать агента

- Задача повторяется регулярно (анализ, проверка, оформление)
- Нужна конкретная экспертиза (бухгалтер, юрист, редактор, аналитик)
- Важно, чтобы AI следовал определённому протоколу (чеклист, порядок действий, формат вывода)
- Над задачей работают несколько ролей, и вы хотите "коллегиальную" работу

Не нужен агент для разовой задачи — достаточно промпта.

## Проверка перед созданием

**Перед созданием любого агента:**

1. Проверь содержимое `~/.claude/agents/` — есть ли файл с таким именем
2. Если файл существует — покажи текущее содержимое и спроси: обновить (с бэкапом .bak), переименовать новый, или оставить как есть
3. Если файла нет — создай как обычно

Это предотвращает случайную перезапись пользовательских агентов при повторном запуске.

## Структура файла агента

```markdown
---
name: имя-агента
description: Одна строка — когда вызывать этого агента
model: opus | sonnet | haiku  # опционально, для контроля бюджета
source: claude-code-practices v1.0  # маркер принадлежности к пакету
---

# Роль
Ты — [роль]. Твоя экспертиза: [области].

# Контекст
[Описание предметной области, терминология, источники данных]

# Правила
1. [Правило поведения]
2. [Правило поведения]
...

# Формат вывода
[Как должен выглядеть результат работы агента]

# Примеры триггеров
- "[фраза пользователя]" → этот агент
```

## Примеры агентов для разных областей

### Финансовый аналитик
**Роль:** анализ P&L, cashflow, налоговых рисков.
**Ключевые правила:**
- Каждая цифра подтверждена расчётом или ссылкой на источник
- Формат вывода: таблицы с итогами, отклонения выделены
- Разделять факты и интерпретации
- При обнаружении аномалий — сигнализировать, а не молча включать в отчёт
**Модель:** opus (нужен глубокий анализ)

### Редактор / корректор
**Роль:** вычитка текстов, стилистика, структура.
**Ключевые правила:**
- Сохранять авторский стиль, не переписывать
- Замечания с обоснованием (не "плохо", а "двусмысленная формулировка, потому что...")
- Разделять критические ошибки (факты, логика) и стилистические
**Модель:** sonnet (процедурная задача)

### Аудитор / ревьюер
**Роль:** проверка документов, отчётов, данных по чеклисту.
**Ключевые правила:**
- Работает по заданному чеклисту, не пропускает пункты
- Выдаёт протокол: что проверено, что найдено, рекомендации
- Разделяет блокеры (нельзя выпускать) и замечания (желательно исправить)
**Модель:** opus (нужна внимательность к деталям)

### Исследователь / аналитик данных
**Роль:** поиск информации, систематизация, подготовка справок.
**Ключевые правила:**
- Указывать источники и степень достоверности
- Структурировать находки по категориям
- Отмечать пробелы в данных ("не удалось найти X")
**Модель:** sonnet

## Коллегиальная работа — несколько агентов на одну задачу

Один AI не может быть одновременно бухгалтером, финдиректором и аудитором. Можно запустить несколько агентов с разными ролями:

**Пример — финансовая аналитика (3 волны):**
1. **Параллельно:** агент-бухгалтер (сводка операций), агент-аналитик (P&L), агент-налоговик (риски)
2. **Синтез:** сопоставление результатов, разрешение противоречий
3. **Оформление:** итоговый отчёт

**Пример — подготовка документа:**
1. **Параллельно:** агент-исследователь (сбор фактов), агент-аналитик (выводы)
2. **Синтез:** агент-редактор (сборка, стилистика, вычитка)

Ключевое: каждый агент видит только свою роль и даёт экспертизу в ней.

## Тиринг моделей

| Модель | Для чего |
|--------|----------|
| opus | Сложный анализ, принятие решений, работа с нюансами |
| sonnet | Процедурные задачи, оформление, работа по шаблону |
| haiku | Простые проверки, извлечение данных, мониторинг |

В сессии с множеством субагентов тиринг экономит бюджет в 3-5 раз.

## Принцип ответственности

Агенты — не слепые исполнители. Если агент видит, что задача некорректна или результат может ввести в заблуждение — он должен сказать об этом. Пропишите это в правилах:

> "Если задача сформулирована некорректно или результат может быть недостоверным — сообщи об этом и предложи уточнение. Необязательное согласие — фича, а не баг."

## Как создавать агентов на лету

Не нужно проектировать заранее. Работаете в сессии, накопили контекст — говорите Claude Code:

> "Создай нового агента `[имя]` на основе нашего текущего контекста. Роль: [роль]. Сохрани в `~/.claude/agents/[имя].md`"

Агент создаётся из накопленного опыта, без перезапуска.
