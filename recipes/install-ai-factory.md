# Как установить и использовать AI Factory

## Что это

AI Factory — open-source инструмент для осознанной разработки с AI. Вместо хаотичных промптов — структурированный конвейер: исследование → планирование → реализация → проверка → обучение на ошибках.

Ключевое отличие от «просто спросить AI»: AI Factory заставляет сначала разобраться в задаче, потом спланировать, и только потом делать. Каждая ошибка записывается и не повторяется.

## Установка

> **Требование:** AI Factory требует Node.js (версия 16+). Если Node.js не установлен — этот пункт можно пропустить, остальные практики работают без него. Установить Node.js: https://nodejs.org/ (выберите LTS-версию).

### Через npm (рекомендуется)
```bash
npm install -g ai-factory
```

### Без установки (разовый запуск)
```bash
npx ai-factory init
```

### Через mise (менеджер версий)
```bash
mise use -g npm:ai-factory
```

## Инициализация в проекте

> **Эту команду нужно выполнить вручную в терминале** — она задаёт вопросы через интерактивное меню (стрелки + Enter). Claude Code не может сделать это автоматически.

Откройте терминал (PowerShell, Terminal) в папке вашего проекта и выполните:

```bash
ai-factory init
```

Вам зададут два вопроса:
1. **Выбор AI-агента** — выберите Claude Code (обычно уже выбран по умолчанию) → нажмите Enter
2. **Выбор навыков** — оставьте все выбранными → нажмите Enter

После этого AI Factory:
- Установит подходящие навыки для вашего стека
- Настроит MCP-серверы (если агент поддерживает)
- Создаст структуру `.ai-factory/` в проекте

После инициализации в проекте появится:

```
.ai-factory/
├── patches/          # Патчи ошибок (самообучение)
├── features/         # Описания функций
├── skills/           # Навыки и знания о проекте
└── workflows/        # Рабочие процессы
```

## Основные команды

### Исследование и планирование

**`/aif-explore`** — перед тем как что-то делать, разберитесь в вариантах:
> /aif-explore Какой подход лучше для авторизации: JWT или сессии?

**`/aif-plan`** — создайте план с задачами, зависимостями и чекпоинтами:
> /aif-plan Добавить систему уведомлений

AI Factory создаст `PLAN.md` с чеклистом задач, точками коммитов и зависимостями.

**`/aif-improve`** — улучшите существующий план:
> /aif-improve Добавь обработку ошибок и edge cases

### Реализация

**`/aif-implement`** — выполнение плана с автоматическими коммитами:
> /aif-implement

AI работает по плану, коммитит на чекпоинтах, отмечает выполненные задачи.

### Исправление ошибок + обучение

**`/aif-fix`** — исправление бага с автоматическим созданием патча:
> /aif-fix TypeError: Cannot read property 'name' of undefined

После исправления создаётся патч-файл в `.ai-factory/patches/`. В следующий раз AI прочитает его и не допустит такую же ошибку.

### Утилиты

**`/aif-docs`** — автогенерация документации
**`/aif-ci`** — генерация CI/CD пайплайнов (GitHub Actions / GitLab CI)
**`/aif-evolve`** — улучшение навыков на основе накопленных патчей

## Как это меняет работу

### Без AI Factory:
```
Промпт → AI пишет код → Ошибка → Ещё промпт → Ещё ошибка → ...
```

### С AI Factory:
```
/aif-explore → Понимание вариантов
/aif-plan → Детальный план
/aif-implement → Реализация по плану с коммитами
/aif-fix → Ошибка записана, больше не повторится
```

## Самообучение через патчи

Каждый `/aif-fix` создаёт патч:

```markdown
# Описание проблемы

Problem: что сломалось
Root Cause: почему
Solution: как починили
Prevention: как предотвратить
Tags: #категория
```

Перед каждой новой задачей AI читает все патчи. Накопленные уроки применяются автоматически. Чем дольше вы работаете с проектом — тем умнее становится AI.

## Совместимость

AI Factory поддерживает множество AI-агентов:
- Claude Code
- Cursor
- Windsurf
- Roo Code
- и другие

Навыки адаптируются под конкретного агента автоматически.

## Совместная работа с Superpowers

AI Factory и Superpowers **дополняют друг друга**, не конфликтуют:

- **Superpowers** меняет *как AI думает* — дисциплина мышления (мозговой штурм, поиск корневой причины, проверка перед «готово»)
- **AI Factory** организует *что AI делает* — конвейер задач (планы, чеклисты, коммиты, патчи)

Аналогия: Superpowers — это характер специалиста (вдумчивый, проверяет за собой). AI Factory — его рабочий стол (план-доска, журнал ошибок, инструменты).

**Рекомендуемый порядок:** сначала Superpowers (работает сразу, в любой сфере), потом AI Factory (когда начнёте работать с конкретными задачами).

Подробнее: `practices/07-ai-factory.md` (секция «Superpowers + AI Factory»).
