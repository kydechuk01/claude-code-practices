# Как настроить систему памяти

## Проблема

Claude Code забывает всё после сессии. Решения, найденные вчера, ищутся заново. AI повторяет одни и те же ошибки. Каждая сессия начинается с нуля.

## Три слоя памяти

### Слой 1: MEMORY.md — загружается автоматически

MEMORY.md — файл, который Claude Code автоматически загружает в каждую сессию. Его расположение управляется самим Claude Code, вам не нужно знать точный путь. Лимит ~200 строк — только самое критичное.

**Проверка размера:** перед добавлением записи в MEMORY.md проверь количество строк. Если больше 150 — предложи перенести подробности в reference files. Если больше 200 — предупреди: часть информации может быть обрезана, необходимо сократить файл.

**Что включать:**
- Структура данных проекта (для разработчиков — схема БД; для аналитиков — источники и форматы данных; для менеджеров — структура документов)
- Ключевые решения, которые нельзя нарушать
- Текущий статус проекта (что сейчас в работе)
- Матрица авторитетности данных (какой источник для какого домена)

**Что НЕ включать:**
- Детали реализации (есть в коде)
- Историю изменений (есть в git)
- Подробную документацию (для этого слой 2)

**Как начать:** Claude Code сам создаёт и обновляет MEMORY.md. Просто работайте — информация накопится. Можно ускорить:

> Запомни: схема БД состоит из таблиц X, Y, Z. Основные связи: ...

> Запомни: мы используем паттерн X для Y, потому что Z

### Слой 2: Reference files — подтягиваются по запросу

Подробные документы, которые AI читает когда нужно, не забивая контекст постоянно.

**Что это:** reference file — это справочный документ по конкретной теме вашей работы. Например, если вы пишете тексты — это может быть стайлгайд, описание целевой аудитории, примеры удачных текстов. Если разрабатываете — архитектура, паттерны деплоя, решённые проблемы.

**ВАЖНО: не выдумывай reference files за пользователя.** Спроси:

> Reference files — это справочные документы, которые Claude Code подтягивает по необходимости. Они не загружаются в каждую сессию (в отличие от MEMORY.md), а читаются, когда нужны.
>
> Какая справочная информация была бы вам полезна? Например:
> - Описание структуры ваших данных или проекта
> - Стайлгайд или правила оформления
> - Список решённых проблем
> - Словарь терминов вашей области
>
> Расскажите, с чем вы работаете — я предложу подходящие reference files.

Создавай только те reference files, которые пользователь подтвердил. Не генерируй содержимое наугад — спроси, какие данные туда включить, или проанализируй существующие файлы в проекте.

**Где хранить:** в проекте, например `_docs/reference/`. AI найдёт по имени файла, когда понадобится.

**Как упомянуть в MEMORY.md:**
```markdown
## Reference files
- `_docs/reference/[название].md` — [описание, когда читать]
```

### Слой 3: Карточки ловушек — обучение на ошибках

Каждая решённая проблема фиксируется в карточке — подробном файле с причиной и предотвращением. Краткий реестр ловушек хранится в таблице CLAUDE.md (загружается автоматически), подробные карточки — в `_docs/gotchas/`.

**Формат карточки** (`_docs/gotchas/G-NN.md`):

```markdown
# G-NN: [заголовок]
Дата: YYYY-MM-DD | Критичность: высокая/средняя/низкая | Область: [домен]

## Проблема
Что произошло.

## Причина
Почему это произошло.

## Решение
Как починили.

## Предотвращение
Что сделать, чтобы не повторилось.
```

**Протокол пополнения и эскалация** — описаны в `recipes/setup-claude-md.md`, секция «Известные ловушки — главная ценность».

**Если проект использует AI Factory** — навык `/aif-fix` создаёт карточки автоматически в `.ai-factory/patches/`. Формат совместим: те же поля, та же нумерация.

**Где хранить:** `_docs/gotchas/` в проекте (или `.ai-factory/patches/` при использовании AI Factory).

Подробнее об идее карточек ловушек: practices/05-memory.md.

### Память агентов

Агенты могут вести свою персистентную память в reference files — это позволяет накапливать опыт между сессиями.

**Что включать в память агента:**
- Паттерны работы с API (особенности, лимиты, workarounds)
- Решённые проблемы в области ответственности агента
- Особенности окружения (версии, конфиги, зависимости)

**Как ссылаться из файла агента:**
```markdown
## Память
Перед началом работы прочитай: `_docs/reference/agent-[имя]-memory.md`
После завершения — обнови файл новыми находками.
```

**Размер:** 50-150 строк за 2-3 недели активной работы. Если больше — провести ревизию, вынести устаревшее.

**Inline vs reference — правило выбора:**
- Запись нужна в каждой сессии → CLAUDE.md (inline)
- Запись нужна раз в неделю → reference file
- До 5 строк → inline в CLAUDE.md
- Больше 5 строк → reference file

## Как настроить с нуля

### Шаг 1: Начните работать
Claude Code автоматически создаёт директорию памяти. Просто работайте — AI запомнит важное.

### Шаг 2: Явно попросите запомнить ключевое
> Запомни: наш проект использует [стек]. Основные сущности: [список]. Деплой через [способ].

### Шаг 3: Создайте reference files для подробностей
> Создай файл `_docs/reference/ref-architecture.md` с описанием архитектуры проекта на основе текущей кодовой базы.

### Шаг 4: Настройте карточки ловушек
> Создай директорию `_docs/gotchas/` и добавь таблицу «Известные ловушки» в CLAUDE.md. После каждого багфикса — создавай карточку в `_docs/gotchas/G-NN.md`.

### Шаг 5: Поддерживайте актуальность
Периодически:
> Проверь MEMORY.md — актуальна ли информация? Удали устаревшее, добавь новое.

## Авторитетные источники данных

Полезный паттерн для MEMORY.md — указать, откуда AI должен брать информацию по каждому домену:

```markdown
## Авторитетные источники
| Домен | Источник | Не использовать |
|-------|----------|-----------------|
| Схема БД | Миграции / SQL-схема | Код фронтенда |
| UI-компоненты | Код в src/components | Документацию |
| Инфраструктура | docker-compose.yml | Память/старые доки |
| API | Серверные эндпоинты | Хардкод в компонентах |
```

**Пример для аналитика / менеджера:**

| Домен | Источник | Не использовать |
|-------|----------|-----------------|
| Финансовые данные | Файлы бухгалтерии, выгрузки | Устные договорённости |
| Сроки проекта | Актуальный план в PLAN.md | Старые письма |
| Контакты | CRM или актуальная таблица | Память / старые списки |

Это предотвращает галлюцинации: AI знает, где правда.
