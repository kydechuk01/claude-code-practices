# Как создавать навыки (skills)

## Что такое навык

Навык — md-файл в `~/.claude/commands/`, вызываемый через `/имя-навыка`. Содержит инструкцию, которую Claude Code выполняет как протокол: чёткие фазы, чеклисты, форматы вывода.

Отличие от агента: агент — это роль ("ты аудитор"), навык — это процедура ("проведи аудит по этому протоколу").

## Проверка перед созданием

Перед созданием навыка проверь `~/.claude/commands/` — есть ли файл с таким именем. Если есть — покажи и спроси: обновить (с бэкапом .bak) или оставить как есть.

## Структура файла навыка

```markdown
# Название навыка — краткое описание

Используй этот скилл когда [триггер].

## Фазы работы

### Фаза 1: [Название]
1. Шаг 1
2. Шаг 2

### Фаза 2: [Название]
...

## Формат вывода
[Шаблон результата]
```

Файл сохраняется как `~/.claude/commands/имя-навыка.md`, вызывается как `/имя-навыка`.

## Навыки, которые стоит создать

### Silent Mode — автономная работа без пользователя

**Проблема:** вы ставите задачу и уходите, но Claude Code постоянно спрашивает разрешения и блокируется.

**Суть навыка — 4 фазы:**
1. **Анализ скоупа** (при пользователе) — понять задачу, определить все нужные ресурсы и инструменты
2. **Запрос разрешений** (при пользователе, до 5 мин) — выполнить по одному реальному действию каждого типа (чтение файла, редактирование, bash-команда...), чтобы пользователь одобрил все permissions до ухода
3. **Автономная работа** (без пользователя) — выполнять задачу, протоколируя каждый шаг в файл; не останавливаться на ошибках — пробовать альтернативы; не задавать вопросов — принимать решения самостоятельно
4. **Финальный отчёт** — статус, что выполнено, блокеры, затронутые файлы, принятые решения

**Ключевые правила:**
- Обязательный файл протокола (лог всех действий с временными метками)
- Красные флаги = СТОП: деструктивные операции, выход за скоуп, потеря данных
- Промежуточные сохранения после логических этапов

**Как создать — скажите Claude Code:**
> Создай навык `/silent-mode` в `~/.claude/commands/silent-mode.md`. Навык для автономной работы без пользователя. Четыре фазы: анализ скоупа, запрос разрешений (прогнать по одному реальному действию каждого нужного типа), автономная работа с протоколированием в файл, финальный отчёт. Красные флаги — деструктивные операции и выход за скоуп. Формат протокола: временная метка + действие + результат.

### Окружение — настройка рабочей среды

**Проблема:** на каждой машине свои пути, версии, особенности. Claude Code не знает, где что лежит.

**Суть навыка:** определить окружение машины один раз и зафиксировать:
- ОС, оболочка, кодировка
- Пути к ключевым инструментам
- Особенности платформы (Windows: пути, кодировка; macOS: homebrew; Linux: apt)

**Как создать:**
> Определи моё окружение: ОС, оболочка, установленные инструменты, пути. Создай навык в `~/.claude/commands/env-setup.md`, который фиксирует это окружение. Навык должен содержать таблицу параметров и правила работы с ними на этой машине.

### Отчёт — оформление результатов работы

**Проблема:** результаты анализа теряются в переписке с AI. Нужен формализованный вывод.

**Суть навыка:**
1. Собрать результаты текущей работы
2. Оформить в заданном формате (markdown, таблицы, секции)
3. Сохранить в файл с датой

**Как создать:**
> Создай навык `/report` в `~/.claude/commands/report.md`. Навык собирает результаты текущей сессии и оформляет в файл. Формат: заголовок, дата, резюме (3-5 пунктов), подробности по секциям, выводы и рекомендации. Сохраняет в `reports/YYYY-MM-DD-[тема].md`.

### Сверка — проверка данных по чеклисту

**Проблема:** ручная проверка данных/документов по списку критериев — рутина, в которой легко пропустить пункт.

**Суть навыка:**
1. Принять объект проверки (файл, данные, документ)
2. Пройти по чеклисту пункт за пунктом
3. Выдать протокол: что проверено, статус, замечания

**Как создать:**
> Создай навык `/check` в `~/.claude/commands/check.md`. Навык для проверки [документов/данных/отчётов] по чеклисту. Чеклист: [перечислить пункты]. Формат вывода: таблица с колонками: пункт, статус (ОК/замечание/блокер), комментарий.

## Как попросить Claude Code создать навык

Общий паттерн:

> Создай навык `/[имя]` в `~/.claude/commands/[имя].md`.
> Назначение: [что навык делает].
> Фазы: [перечислить этапы].
> Формат вывода: [как должен выглядеть результат].
> Правила: [ограничения и красные флаги].

Claude Code создаст файл, и навык сразу будет доступен через `/имя`.

## Активация навыков — как повысить вероятность вызова

**Проблема:** установленные навыки часто не запускаются автоматически. Claude Code вызывает навык только когда пользователь явно набирает `/имя-навыка`. При обычных задачах модель «забывает» проверить, подходит ли установленный навык.

**Причина:** вызов навыка — решение модели, а не автоматический триггер среды. Модель должна сама сопоставить задачу со списком навыков и вызвать Skill tool. На практике это происходит нестабильно, особенно в длинных сессиях.

### Уровни надёжности активации *(техническая справка)*

| Уровень | Механизм | Вероятность |
|---------|----------|-------------|
| **Гарантия** | Hooks в settings.json | ~99% — выполняет среда, не модель |
| **Высокая** | Пользователь пишет `/команда` | ~95% |
| **Средняя** | Явный маппинг в CLAUDE.md | ~50-70% |
| **Низкая** | Описание навыка в каталоге | ~10-20% |

### Маппинг в CLAUDE.md — простой способ поднять вероятность

Добавьте в CLAUDE.md вашего проекта блок с явным маппингом «задача → навык». Пример для проекта с AI Factory:

```markdown
## Навыки — ОБЯЗАТЕЛЬНО

**CRITICAL: Перед началом работы определи тип задачи и вызови навык:**

| Тип задачи | Навык |
|------------|-------|
| Новая фича / крупная задача | `/aif-plan` → `/aif-implement` |
| Исследование вариантов | `/aif-explore` |
| Баг / ошибка | `/aif-fix` |
| Улучшение существующего плана | `/aif-improve` |
| Code review | Навык ревью или `/aif-improve` |
| Автономная работа без пользователя | `/silent-mode` |

**НЕ НАЧИНАЙ работу без проверки этой таблицы. Это НЕ рекомендация.**
```

**Почему работает лучше, чем просто установить навыки:**
- **CRITICAL** — повышает приоритет инструкции в контексте модели
- **Таблица, а не текст** — модель быстрее находит и парсит
- **«НЕ НАЧИНАЙ»** — явный запрет действовать без проверки
- **Конкретный маппинг** — модели не нужно гадать, какой навык подходит

Адаптируйте таблицу под ваши навыки. Если используете не AI Factory, а собственные — замените команды на свои.

**Ловушка:** навыки в маппинг-таблице должны быть реально установлены. Если таблица содержит `/aif-plan`, а AI Factory не установлен — Claude Code попытается вызвать несуществующий навык и потеряет контекст. Перед добавлением навыка в маппинг — убедитесь, что он доступен.

### Ограничения — честное предупреждение

Даже с маппингом в CLAUDE.md навыки **не срабатывают в 100% случаев**. Это фундаментальное ограничение: CLAUDE.md — инструкция для модели, а не исполняемый код. Модель может проигнорировать инструкцию, особенно:
- В длинных сессиях (инструкции «размываются» с ростом контекста)
- При нечётком типе задачи (граница между «фичей» и «багом» размыта)
- На менее мощных моделях (Haiku следует инструкциям слабее, чем Opus)

**Правило:** для workflow, где пропуск навыка критичен — используйте хуки (hooks) в settings.json, а не маппинг в CLAUDE.md. Хуки выполняются средой исполнения, а не моделью, и работают со стабильностью ~99%.

## Навык vs промпт

Если вы ловите себя на том, что каждый раз пишете одну и ту же длинную инструкцию — это кандидат в навык. Навык экономит время и гарантирует, что ничего не забыто.
