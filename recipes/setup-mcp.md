# Как подключить MCP-серверы

> Этот рецепт требует базовых навыков работы с терминалом. Если вы не разработчик — попросите Claude Code установить MCP-серверы за вас. Технические детали ниже — для тех, кто знаком с командной строкой.

## Что это

MCP (Model Context Protocol) — расширения, которые дают Claude Code доступ к внешним системам: документации библиотек, браузеру, базам данных, GitHub. Без них AI работает только с файлами на диске. С ними — может проверить актуальную документацию, открыть сайт, сделать скриншот, выполнить SQL-запрос.

MCP-серверы — это не плагины Claude Code. Это отдельные процессы (локальные или удалённые), которые Claude Code вызывает по протоколу. Один сервер = один набор инструментов.

## Два рекомендуемых сервера для старта

### Context7 — актуальная документация библиотек

**Зачем:** Claude Code обучен на данных до определённой даты. Библиотеки обновляются, API меняются. Без Context7 AI может предложить устаревший метод или несуществующий параметр. С Context7 — подтягивает документацию конкретной версии в реальном времени.

**Пример:** спрашиваешь «как сделать X в React Query v5» — Claude Code обращается к Context7, получает актуальную документацию и отвечает на её основе, а не по памяти.

**Рекомендация:** установить глобально. Это самый полезный MCP-сервер для предотвращения галлюцинаций о библиотеках.

### Playwright (Browser) — управление браузером

**Зачем:** AI может открыть веб-страницу, сделать скриншот, нажать кнопку, заполнить форму — всё автоматически.

**Visual QA Loop — главный паттерн:**
```
Изменение → Сборка → Скриншот → Визуальная проверка AI → Итерация
```

AI сам проверяет результат визуально: правильно ли выглядит страница, не сломалась ли вёрстка. Без участия человека.

**Применение за пределами разработки:**
- Скриншоты сайтов для анализа (конкуренты, источники)
- Автоматическое заполнение веб-форм
- Извлечение данных со страниц

## Установка

### Вариант 1 — из Claude Code (рекомендуемый)

Скажи Claude Code:

> Установи MCP-сервер Context7 для работы с документацией библиотек.

Или:

> Установи MCP-сервер Playwright для управления браузером.

Claude Code выполнит установку и покажет результат.

### Вариант 2 — из терминала (CLI)

```bash
claude mcp add context7 -- npx -y @upstash/context7-mcp
```

```bash
claude mcp add playwright -- npx -y @playwright/mcp@latest
```

**Проверка:** после установки выполни в Claude Code `/mcp` — сервер должен быть в списке.

### Вариант 3 — через файл .mcp.json (для команды)

Создай файл `.mcp.json` в корне проекта:

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    },
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
```

Этот файл можно закоммитить в git — все участники проекта получат те же MCP-серверы автоматически.

## Если установка не удалась

**«npx не найден»** — нужен Node.js. Установи с https://nodejs.org/ (LTS версия). После установки перезапусти терминал.

**«Connection refused» или сервер не отвечает** — выполни `claude --mcp-debug` для диагностики. Часто помогает перезапуск Claude Code.

**Сервер установлен, но инструменты не появляются** — MCP-инструменты загружаются по требованию (on-demand). Они появятся, когда Claude Code решит, что они нужны для текущей задачи. Проверить список: `/mcp`.

## Другие полезные серверы

| Сервер | Что даёт | Команда установки |
|--------|---------|-------------------|
| GitHub | PR, issues, push из Claude Code | `claude mcp add github` (требует токен) |
| PostgreSQL | Прямые SQL-запросы к базе | `claude mcp add postgres -- npx -y @anthropic/mcp-postgres` |
| Supabase | Полная интеграция с Supabase | Через HTTP: `https://mcp.supabase.com/mcp` |
| GitNexus | Граф знаний кодовой базы, impact analysis | `npx gitnexus analyze` |

Полный список серверов: https://github.com/modelcontextprotocol/servers

## Глобально vs. на проект

- **Глобально** (`claude mcp add --global`) — сервер доступен во всех проектах. Подходит для Context7, GitHub.
- **На проект** (`.mcp.json` в корне) — сервер работает только в этом проекте. Подходит для PostgreSQL (разные базы в разных проектах), специфичных серверов.
