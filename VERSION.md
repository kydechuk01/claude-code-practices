## Changelog

### 1.7 (2026-04-14)
- Новый рецепт: `session-handoff.md` «Сеансовая эстафета» — передача контекста новой сессии через самодостаточный документ с гарантированной доставкой ядра §0+§1 через SessionStart hook. Рецепт построен на анализе трёх независимых рабочих практик из разных доменов (разработка, документная подача, бухгалтерия/аналитика) — все три эмпирически пришли к одной архитектуре mandatory-trigger.
- Новое правило 7 в `behavior-rules.md` «Надёжность доставки критичных инструкций» — границы применимости CLAUDE.md, обоснование SessionStart hook как механизма за пределами «совета в тексте». Эмпирическая база — сходимость трёх проектов плюс контролируемый A/B-тест.
- Главный принцип эстафеты: «не отчёт о сделанном, а исполнимая программа холодного старта». Подкреплено пятью операционными правилами: TL;DR пишется последним, цифры live-verified, готово а не процесс, mandatory-trigger через hook, invariant-pointer LATEST вместо glob-поиска.
- Восемь обязательных секций (§0/§1/§2/§3/§4/§5/§6/§11) — универсальны для любого домена. Опциональные расширения (параллельные ветки, EDIT-LIST, карточки стейкхолдеров, multi-agent сборка) — по профилю проекта, со ссылками на companion-рецепты.
- Hook-формула обновлена на invariant-pointer: `head -n 80 docs/SESSION-HANDOFF-LATEST.md` вместо glob по дате. Снимает риск двусмысленности имён и не требует правок CLAUDE.md при каждом новом handoff.
- Лимит hook-output 10 000 символов — задокументирован, ядро §0+§1 рассчитано на гарантированное прохождение.
- Граница между `track-progress.md` (маркер «где я») и `session-handoff.md` (полный снимок) чётко разведена, явный алгоритм выбора, cross-link из track-progress.
- Обновлён `parallel-agents-watchdog.md`: Python-код шага 4 заменён на псевдокод, контрольные вопросы перенесены в начало, шаблоны брифов вынесены в Appendix, раздел интеграции обновлён на «сеансовая эстафета» с правилом «synthesis stays in main thread».
- Добавлен пункт 2.5.1 в `START.md` со сравнительной таблицей эстафеты и track-progress, главным принципом и базовыми триггерами применения.
- **Команды активации в обычной речи:** «давай сделаем хэндофф» / «собери эстафету» / «собери session handoff» — без slash-команд и точной формулировки. Секция «Как вызвать» в начале рецепта, упоминание в блоке CLAUDE.md и в START.md.
- **Уровни настройки практики (0/1/2):** базовая регистрация в CLAUDE.md без файлов и hook (Уровень 0, по умолчанию) → первая эстафета по запросу через 5 вопросов (Уровень 1) → SessionStart hook с проверками `jq`/существующих hook (Уровень 2). Делает поведение Claude Code предсказуемым на этапе настройки.
- **Строгие правила «НЕ»** для Claude Code при настройке: не создавать эстафетный файл сразу, не настраивать hook без явного запроса, не заполнять §0 за пользователя по догадкам, не путать эстафету и track-progress.
- 17 рецептов (было 16)

## Процедура обновления

При переходе на новую версию пакета:
1. Скачайте новую версию
2. Запустите START.md — он обнаружит метки старой версии в CLAUDE.md
3. Покажет изменённые секции и спросит подтверждение
4. Обновит помеченные секции, не трогая пользовательские
